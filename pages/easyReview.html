Easy Review

Prototype app

Overview

  • Goal: To build a prototype app for easy and quick restaurant review system.
  • Duration: Oct’13-Dec’13
  • Users: Smartphone users
  • My Role: User Researcher and User Experience Designer
  • Tools Used:Pen and Paper, Balsamiq, JustInMind
  • Design Process: User Research -> Competitor Analysis -> Story board -> Wireframe->Prototype

Action

Need Finding:

My researching methods was a combination of interviews and user observation sessions.This is because, a sandwich of observation technique between two interview sessions helped me to dig deeper into the user needs.

  • Asking participants to answer few general questions at the beginning to set the stage.
  • Next observing them perform a task of reviewing a restaurant for the last visited one.
  • In addition to this, a final interview session addressing specific questions to explore the experience they just had performing the task worked out well to identify user needs and find design opportunities for innovation.

Competitor Analysis:

Following user research, I explored existing reviewing system such as yelp, amazon, google review, seamless and etc, that users frequently use to either read or write a review. The main aim of this process was to get inspiration from already existing services at the same time to consider a totally different path that others did not take if necessary. This session was a sort of competitor analysis cum ideation which set the stage for storyboarding.

Story boarding:

Creating a scenario helped me to dive deeper into the design problem. Few things like mentioned below were identified

  • Who are my user?
  • When will this app be used?
  • What can motivate the user to use the app?
  • What will the user get in return for using my app?

Prototyping:

After brainstorming with story boarding, next step was to turn the sketches into experience. After careful analysis of each design solutions, I came up with a design that connected the dots and fulfilled the user needs. The final prototyping processes was iterative with steps like Design, Review, and Refine in every iteration.

Design

To picture how the app will look and behave, I rapidly designed a low fidelity yet interactive prototype of the entire work flow. This wireframe offered the basic functionality of the application and this was designed using ‘Balsamiq' wire framing tool.

Review

The interactive wireframe was then reviewed by a set of peer designers. The reviewers conducted Heuristic Evaluation of interactive prototype to find usability bugs using Nielson’s 10 heauristics.

Refine

This step involved listing all the usability issues from heuristic evaluation and improving on it as needed. This high fidelity prototype of EasyReview app designed using the prototyping tool ‘JustinMind’.

Sample Heuristic Evaluation:

Prototype 1:

User control and freedom: A user can write an anonymous review but nowhere is the user warned about the implications of being anonymous. The implication is that the reward (10% discount) is not available (this is what I inferred from the "Welcome back" text on page 6). Severity: 3 = Major usability problem: important to fix, so should be given high priority

Error prevention: It is only *after* a user has posted as anonymous is the user told that writing the review after signing in would have earned an award. There is no way for an anonymous user to immediately change an anonymous review into a signed-in one. Severity: 3 = Major usability problem: important to fix, so should be given high priority

Error prevention: A user is asked to grade between a scale of 1 to 5 but there is no indication if 1 means bad or good. In some cultures, 1 might mean "best". Some sort of an indication is needed to tell users which number is the lowest in the grading scale. Severity: 4 = Usability catastrophe: imperative to fix this before product can be release

User Testing:

After developing a fully functional prototype, it was time to test it with the real users. This was the second iteration of reviewing the design. User testing was very important at this stage to observe and get insights of the experience the app offered to the end users. To manage the user tests properly and to get maximum feedback out of it, I developed user evaluation plan and user test scripts. I also identified 3 participants to conduct user testing. Redesign:

After testing, an elaborate analysis on the recordings and observation notes were done to find if the goal of the app has satisfied the user’s need. A behavior pattern was defined and list of changes to the current design was identified to implement in the next phase of the project.

Key Learnings

Iterative Design and Development:

As a budding UX designer, I identified a behavioral problem and devised a design solution to enhance user experience. I understood, the solution involves steady refinement of the design based on user testing and other evaluation methods.

For example:

In this project, users can rate certain qualities of the restaurant with a rating scale 1 to 5 points (Radio buttons). In the first iteration, the rating scale did not visually represent its respective values i.e. does scale 5 supposed to mean extremely bad or extremely good? However, one of the early heuristic evaluation revealed a potential usability assumption or mental model mismatch if you will and helped me to work on it in the next iteration.

Improvements

Even though the user flow and experience of the app is well crafted, the UI still need lots of improvement. Few things are:

  • Aesthetic aspects of the interface.
    • Removing conspicuous background image in every page of the final prototype.
  • Follow inspirational mobile UI design patterns.
    • Hamburger Navigation, layouts and grids, etc.
  • Specific design principles
    • Mobile friendly error message display
  • Succinct Labels and Languages used.
